# 06 — Admin Panel

The admin panel is the product's most important feature. The owner is a
jewellery manufacturer, not a developer — anything they might want to change
more than once must be changeable here, in plain English, without a deploy.

**Access:** `/admin` — requires a logged-in user with `isAdmin: true`.
Non-admins are sent to `/login?callbackUrl=…` (a prompt, not a silent bounce).

---

## 1. Layout

- **Dark slate sidebar** (`#1e293b`) — deliberately a different surface from the
  cream storefront, so the owner always knows which side they're on.
- **Mobile:** hamburger at top-left, slide-in drawer, backdrop, close button.
  The whole panel is usable from a phone.
- **Quick nav** (`AdminQuickNav`) — a "tell me what you want to do" launcher at
  the top of the sidebar.
- **Live badges** — Pending count and unread Messages count render in the
  sidebar. **Pending sits at the top of the nav while there's work to do and
  drops to the bottom when the count is zero.**
- **Footer link** back to the live store.

### Sidebar items

Dashboard · *Pending (when non-zero)* · Orders · Messages · Products ·
Categories · Shop · Colors · Coupons · Hero Section · Header Ribbon · Blog ·
Settings · Admins · Help · My Account · *Pending (when zero)*

---

## 2. Dashboard — `/admin`

At-a-glance stats: total orders, revenue (formatted `en-IN` ₹), trend, and
recent orders with colour-coded status pills (Processing amber / Shipped blue /
Delivered green / Cancelled red).

---

## 3. Pending — `/admin/pending`

A checklist of catalogue chores generated by querying for incomplete data:
products missing images, terms missing cover images or blurbs, products not yet
assigned to taxonomy terms, and so on. Each item shows a count, a one-line
explanation of why it matters, and a link straight to the screen that fixes it.

Every check is individually wrapped, so one failing query degrades that item to
zero rather than taking down the admin.

This exists because tagging a 191-product catalogue is a long job — the checklist
turns it into a finite, visible queue.

---

## 4. Orders — `/admin/orders`, `/admin/orders/[id]`

- Full order list.
- Detail view: line items, customer name/email/phone, shipping address, notes,
  totals, discount, coupon used, payment method and payment status.
- **Status dropdown** — Processing → Shipped → Delivered, or Cancelled. Saves
  immediately.

---

## 5. Messages — `/admin/messages`

Every contact-form submission, with an unread badge in the sidebar.

- Messages are written to the database **first** — the notification email is
  best-effort on top, so no message is ever lost while email is unconfigured or
  down.
- Click the customer's email to reply from your mail app.
- Mark read / delete.

---

## 6. Products — `/admin/products`

List with a search box, plus `/admin/products/new` and
`/admin/products/[id]/edit`.

The product form covers:

| Field | Notes |
|---|---|
| Name | Slug auto-fills from it |
| Slug | Editable, unique |
| SKU | Unique. An **"Auto" button auto-detects the category from the SKU code** |
| Categories | Tick **one or more**; the first is treated as primary |
| Material | |
| Price | |
| Compare-at price | Renders struck-through next to the price |
| Stock | `0` = out of stock |
| Images | Click-to-upload or drag-and-drop, max 5 MB each, uploaded to Cloudinary. **First image is the cover**; hover to set cover or remove |
| Badge | Chosen from the owner's own badge list |
| Occasions / styles / tags | Multi-select |
| Featured | Shows in the homepage Featured section |
| Front-page position | Integer; `1` = first. Pinned products always precede unpinned. Blank = unpinned |
| Meta title / meta description | Optional per-product SEO overrides; fall back to auto-generated |

Saving triggers **on-demand revalidation** of the affected pages, so a change is
live immediately without waiting for an ISR window.

---

## 7. Categories — `/admin/categories`

The simple homepage-card category list: name, slug (auto-filled), image (URL or
browse-to-upload), sort order, show-on-home toggle. Edit and delete inline.

Deleting a category removes the card; products keep their category tag.

---

## 8. Shop (taxonomy) — `/admin/shop`

The powerful one. Manages the whole **multi-dimension taxonomy**.

- Create, rename, reorder, hide **groups** ("Shop by ___" dimensions). The six
  system groups are Category, Occasion, Collection, Look, Stone, Colour — and
  the owner can add new ones.
- Only the Category group is hierarchical (main → sub).
- Per **term**: label, slug, blurb, cover image + focal point, hex colour, sort
  order, show-in-menu toggle, parent (for hierarchical groups).
- Terms automatically drive the mega-menu, the shop sidebar filters, the
  homepage "shop by" rows, and indexable facet URLs. Adding a dimension needs no
  code.

---

## 9. Colors — `/admin/colors`

A focused editor for the `colour` taxonomy group — term label, slug, hex swatch,
sort order, menu visibility. Split out from the Shop screen because colour terms
need a swatch picker rather than an image.

---

## 10. Coupons — `/admin/coupons`

- Code (auto-uppercased), type (**percentage** or **flat ₹**), value.
- Optional minimum order amount, max uses, expiry date.
- Active toggle — turn a coupon off without deleting it.
- Usage count shown.
- Machine-minted lead coupons (those with `issuedToEmail`) are filtered out of
  this list so the owner only sees coupons they created.
- Toggling active no longer truncates the expiry date (a real bug that was fixed).

---

## 11. Hero Section — `/admin/hero`

The most-used screen after Products.

- Upload one or more slides; multiple active slides become a **crossfade
  slideshow**.
- **Rotation speed** in seconds (clamped 1.5s–30s).
- Per slide:
  - Optional **separate mobile image**.
  - **Click-to-set focal point** on a Desktop preview *and* independently on a
    Mobile preview — this is how the owner "crops" a photo per device without
    editing the file.
  - Brightness, contrast, overlay opacity.
  - Reorder arrows, eye icon to hide/show, bin to delete.
- If there are no slides at all, the storefront falls back to a shipped default
  hero image, so the homepage can never break.

---

## 12. Header Ribbon — `/admin/ribbons`

Edit the rotating announcement messages at the very top of the site. Add,
reorder, remove, save. Empty/whitespace entries are filtered out and an empty
list falls back to the three shipped defaults.

---

## 13. Blog — `/admin/blog`

Create and edit Journal articles: slug, title, excerpt, cover image, structured
body (an array of `{ heading?, paragraphs[] }`), related links, read time,
published toggle, meta title/description, publish date.

Body is structured — never raw HTML — so owner-written content can't inject
markup.

---

## 14. Settings — `/admin/settings`

A hub of seven cards.

### 14a. Business details — `/admin/settings/business`
The single source of truth for: public email, phone, WhatsApp number,
Instagram URL + handle + follower text, address line, city, region, postal code,
country code, opening hours, JustDial URL.

Read by the Footer, Contact page, WhatsApp button, Organization JSON-LD,
LocalBusiness JSON-LD and the Instagram strip — so it's edited once.

### 14b. Homepage — `/admin/settings/homepage`
- **Promo banner** — enabled, text, CTA label, CTA href, background colour.
  Ships **disabled** so it doesn't appear until turned on.
- **Trust badges** — the strip under the hero. Each is an icon (from a fixed
  set of 10 Lucide icons: shield, truck, exchange, lock, badge, gem, heart,
  award, sparkles, gift), a title and a sub-line. Defaults are the five shipped
  badges.
- **Testimonials** — sourced from real published reviews; optionally curate
  specific review IDs and their order. Falls back to 15 shipped testimonials if
  there are none.
- **Brand story** — heading, body, CTA label, CTA href, image.
- **Pull quote** — text + attribution.
- **Section order & visibility** — reorder and toggle all 12 post-hero sections.
  Unknown keys are dropped; newly shipped sections are appended automatically,
  enabled.

### 14c. Pricing & shipping — `/admin/settings/commerce`
| Setting | Default | Meaning |
|---|---|---|
| GST rate | `0.03` (3%) | Stored as a fraction; values > 1 are rejected |
| Gift-wrap fee | ₹49 | |
| Shipping fee | ₹0 | Flat fee charged below the threshold |
| Free-ship threshold | ₹0 | `0` disables the check → shipping stays free |
| COD enabled | `true` | |
| COD max order | ₹0 | `0` = no cap |

These feed `computeTotals()`, which is imported by the checkout form **and**
both checkout API routes.

### 14d. Products & catalog — `/admin/settings/catalog`
- **Badges** — the owner's own list of label + Tailwind colour-class pairs.
  Unknown labels fall back to a grey pill.
- **Low-stock threshold** (default 5) — when the PDP shows "only N left".
- **Hide out-of-stock** (default off).
- **Default sort** — newest / price ↑ / price ↓ / name A–Z.

### 14e. Pages & FAQ — `/admin/settings/content`
Edit the About, Shipping, Privacy and Terms pages (title, intro, sections of
heading + body, "last updated"), and the full FAQ list (question + answer).

Also holds the **shared shipping timing** — delivery days, refund days, return
days — set once and interpolated into the Shipping page, the FAQ and the
structured data. This exists because those numbers had drifted apart across
pages.

### 14f. Navbar — `/admin/settings/navbar`
- Announcement bar background + text colour, header background, accent colour.
- Link list: label, href, visibility, order, and **type** —
  `link` (plain anchor), `megamenu`, `occasion`, or `collection` (each rendering
  its own dropdown component).
- Colours pass the same sanitiser as the theme editor; an invalid or empty link
  list falls back to the nine shipped defaults.

### 14g. Theme & fonts — `/admin/settings/theme`
The five brand colours (see [05 — Design System](05-design-system.md)) and the
five font pairings, with a live preview. An untouched theme emits no CSS at all.

---

## 15. Admins — `/admin/admins`

- Add an admin: username, optional display name, password.
- Edit another admin's username or password.
- Delete — **blocked for yourself and for the last remaining admin.**
- Usernames are stored lowercase and matched case-insensitively.

---

## 16. Help — `/admin/help`

A searchable how-to library written in the owner's own language, referring to
the sidebar labels they actually see. Every topic has:

- a group (Products, Categories, Coupons, Orders, Home page, Admins, My account)
- extra search keywords (so "sold out" finds the stock topic)
- numbered steps
- a **"Tell Me" deep link** that opens the exact admin screen for that task

Topics currently cover: adding a product, editing/deleting a product, putting a
product in multiple categories, product images and cover, featuring a product,
front-page position, stock / out-of-stock, sale prices, adding and editing
categories, adding/editing/deactivating coupons, viewing and updating orders,
reading and replying to messages, changing hero images, per-device hero
cropping, editing the ribbon, adding an admin, changing another admin's
credentials, and changing your own account.

**This file (`lib/adminHelp.ts`) must be updated whenever an admin screen
changes.** It is the owner's manual.

---

## 17. My Account — `/admin/account`

Change your own username, display name, or password. The **current password is
required** to save any change.
