What's New vs Current

New Capabilities

  • Visual crop tool — drag a rectangle to define the visible area for desktop (16:9) and mobile (9:16) separately. Corner handles to resize.
  • Replace image inline — "Replace Image" button swaps the image without deleting the slide (preserves order, active state, etc.)
  • Full-size live preview — see exactly how the hero renders on desktop and mobile, with the site's actual overlay/gradient applied
  • Basic adjustments — brightness, contrast, overlay darkness sliders. CSS filters applied at render time (no image re-processing needed)
  • Separate mobile image — keep the existing "use different image on mobile" option

Implementation Notes

  • Crop is CSS-based (object-position + object-fit) — no server-side image processing needed
  • Adjustments stored as CSS filter values on the HeroSlide model
  • Replaces the current simple focal-point-click UI entirely
  • New DB fields: cropDesktop, cropMobile, brightness, contrast, overlayOpacity
